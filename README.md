## О проекте

**ChatApp** — учебный проект дисциплины «Современные способы программирования» (магистратура, 2026).
Реализован в двух вариантах:

| Версия | Папка | База данных | Порт | Описание |
|--------|-------|-------------|------|----------|
| SQLite | `sqlite_version/` | SQLite (локальный файл `chat.db`) | **5001** | Групповой чат, запуск без настройки |
| MySQL  | `mysql_version/`  | MySQL (удалённый сервер) | **5000** | Личные переписки 1-на-1, полный MVP |

Оба приложения можно запускать одновременно — они работают на разных портах.

---

## Возможности

- Регистрация с именем, фамилией, логином и паролем (пароль хешируется через `werkzeug`)
- Авторизация по логину и паролю; сессия сохраняется между страницами
- Список зарегистрированных пользователей для выбора собеседника
- Личный чат 1-на-1: история переписки хранится в базе данных
- Динамическая отправка и получение сообщений без перезагрузки страницы (fetch API + polling)
- Пузырьки сообщений: свои — справа, чужие — слева
- Группировка сообщений по датам
- Аватарки с инициалами пользователей и цветовым кодом
- Поиск по контактам в сайдбаре
- Адаптивный интерфейс (работает на телефоне)
- Защита маршрутов: страницы недоступны без авторизации

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/MavrinIlua/chat_project.git
cd chat_project
```

### 2. Создать одно виртуальное окружение для обоих приложений

```bash
python -m venv venv
```

Активировать:

```bash
# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

Установить все зависимости сразу:

```bash
pip install flask werkzeug mysql-connector-python python-dotenv
```

> Одного виртуального окружения достаточно для обоих приложений — зависимости у них общие.

---

### 3. Запустить SQLite-версию (порт 5001)

```bash
cd sqlite_version
python app.py
```

Открыть: [http://127.0.0.1:5001](http://127.0.0.1:5001)

> База данных `chat.db` создаётся автоматически в папке `sqlite_version/` при первом запуске.

---

### 4. Запустить MySQL-версию (порт 5000)

Сначала создать файл с настройками подключения:

```bash
cd mysql_version
cp .env.example .env   # macOS/Linux
copy .env.example .env # Windows
```

Открыть `.env` и заполнить (см. раздел ниже), затем:

```bash
python app.py
```

Открыть: [http://127.0.0.1:5000](http://127.0.0.1:5000)

> Таблицы в MySQL создаются автоматически при первом запуске.

## API

MySQL-версия реализует REST-подобный JSON API:

| Метод | Маршрут | Тело (JSON) | Описание |
|-------|---------|-------------|----------|
| `POST` | `/api/register` | `name, surname, login, password` | Регистрация пользователя |
| `POST` | `/api/login` | `login, password` | Авторизация |
| `GET`  | `/api/messages/<partner_id>` | — | История переписки |
| `POST` | `/api/messages/<partner_id>` | `text` | Отправить сообщение |

Пример успешного ответа:

```json
{ "ok": true, "user_id": 7 }
```

Пример ответа с ошибкой:

```json
{ "ok": false, "message": "Логин уже занят" }
```

## Деплой

Чтобы приложение было доступно онлайн по публичной ссылке, используют хостинг.
Самый простой бесплатный вариант для Flask — **Render**.

### Деплой на Render (бесплатно)

1. Зарегистрироваться на [render.com](https://render.com)
2. New → **Web Service** → подключить репозиторий `MavrinIlua/chat_project`
3. Настройки:
   - **Root Directory:** `mysql_version`
   - **Runtime:** Python 3
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
4. В разделе **Environment** добавить переменные из `.env` (`DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `SECRET_KEY`)
5. Нажать **Deploy** — через ~2 минуты приложение будет доступно по адресу вида `https://chat-project-xxxx.onrender.com`

Также установить `gunicorn` в зависимости:

```bash
pip install gunicorn
# и добавить в requirements.txt:
echo "gunicorn" >> mysql_version/requirements.txt
```
