## О проекте

**ChatApp** — учебный проект дисциплины «Современные способы программирования» (магистратура, 2026).  
Реализован в двух вариантах:

| Версия | Файл | База данных | Описание |
|--------|------|-------------|----------|
| `sqlite_version` | `app.py` | SQLite (локально) | Групповой чат, простой запуск без настройки |
| `mysql_version`  | `app.py` | MySQL (удалённый сервер) | Личные переписки 1-на-1, полный MVP |

---

## Возможности

- Регистрация с именем, фамилией, логином и паролем (пароль хешируется через `werkzeug`)
- Авторизация по логину и паролю; сессия сохраняется между страницами
- Список зарегистрированных пользователей для выбора собеседника
- Личный чат 1-на-1: история переписки хранится в базе данных
- Динамическая отправка и получение сообщений без перезагрузки страницы (fetch API + polling)
- Пузырьки сообщений: свои — справа, чужие — слева
- Группировка сообщений по датам
- Аватарки с инициалами пользователей
- Поиск по контактам
- Адаптивный интерфейс (работает на телефоне)
- Защита маршрутов: недоступны без авторизации

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/<ваш-логин>/chat_project.git
cd chat_project
```

### 2. Выбрать версию и создать виртуальное окружение

```bash
cd mysql_version          # или sqlite_version
python -m venv venv
```

Активировать:

```bash
# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить переменные окружения (только для MySQL-версии)

```bash
cp .env.example .env
```

Открыть `.env` и заполнить своими данными подключения к MySQL (см. раздел ниже).

### 5. Запустить приложение

```bash
python app.py
```

Открыть в браузере: [http://127.0.0.1:5000](http://127.0.0.1:5000)

> Таблицы в базе данных создаются автоматически при первом запуске.

## API

MySQL-версия реализует REST-подобный API:

| Метод | Маршрут | Тело (JSON) | Описание |
|-------|---------|-------------|----------|
| `POST` | `/api/register` | `name, surname, login, password` | Регистрация пользователя |
| `POST` | `/api/login` | `login, password` | Авторизация |
| `GET`  | `/api/messages/<partner_id>` | — | История переписки |
| `POST` | `/api/messages/<partner_id>` | `text` | Отправить сообщение |

Все эндпоинты возвращают JSON. Пример успешного ответа:

```json
{ "ok": true, "user_id": 7 }
```

Пример ответа с ошибкой:

```json
{ "ok": false, "message": "Логин уже занят" }
```
